# Vision Utilities

**Vision Utilities** is a **ROS Noetic** package and a submodule of the **Perception Module** in the **SinfonIA** workspace. It provides essential vision-related functionalities and services for the perception system, including support for QR code recognition, object detection, pose estimation, and more.

## Installation & Setup

To use this package, you must clone it into a ROS Noetic workspace and install the specific Python dependencies based on your use case.

### 1. Clone the Repository

Navigate to the `src` folder of your catkin workspace and clone the repository using SSH:

```bash
cd ~/catkin_ws/src
git clone git@github.com:SinfonIAUniandes/vision_utilities.git

```

### 2. Install Dependencies

This repo contains three different requirement files. Choose the one that fits your needs and install it via `pip`:

* **All Dependencies:** Installs every requirement for all features.
```bash
pip install -r all_requirements.txt

```


* **AI Features:** Includes MediaPipe, Ultralytics, and DEAP for vision tasks.
```bash
pip install -r requirements_ai.txt

```


* **Basic Usage:** Only core functionalities (no AI/Heavy models).
```bash
pip install -r requirements_basic.txt

```



### 3. Build the Workspace

Return to the root of your workspace and build the package:

```bash
cd ~/catkin_ws
catkin_make
source devel/setup.bash

```

## External Camera Setup (`usb_cam`)

Instead of a custom publisher, we use the standard `usb_cam` package to stream video from local webcams.

### Installation

Install the package via the official ROS binaries:

```bash
sudo apt update
sudo apt install ros-noetic-usb-cam

```

### Usage

To start your webcam and publish frames to the ROS network:

```bash
# Basic start (usually /dev/video0)
rosrun usb_cam usb_cam_node

# Start with specific parameters (e.g., pixel format or device)
rosrun usb_cam usb_cam_node _video_device:="/dev/video0" _pixel_format:="yuyv"

```

*The node publishes frames to `/usb_cam/image_raw` by default.*

## Configuration (`config.yaml`)

The primary way to configure the module is through the `config.yaml` file located in `vision_utilities/src/config.yaml`. This file allows you to toggle robot modes, AI services, and model parameters without changing code.

### Configuration Overview

| Section | Parameter | Type | Default | Description |
| --- | --- | --- | --- | --- |
| **Hardware** | `with_pepper` | Boolean | `false` | **Robot Mode:** Set to `true` to enable compatibility with the Pepper robot; `false` uses local devices (e.g., `usb_cam`). |
|  | `start_cameras` | Boolean | `false` | **Auto-Start:** If `true`, the module attempts to initialize and open the camera stream automatically upon startup. |
| **VLM** | `llm_mode` | String | `"openai"` | **Backend Provider:** Defines the Vision-Language Model host. Options are `"openai"` (Cloud) or `"ollama"` (Local). |
|  | `model` | String | `"GPT-4o"` | **Model ID:** The specific VLM to use (e.g., `GPT-4o`, `gemma3_4b`, `gemma3_12b`). |
|  | `max_tokens` | Integer | `500` | **Response Limit:** The maximum length of the text description generated by the VLM. |
| **YOLO** | `model_name` | String | `"yolo11n"` | **Object Detection:** The YOLO model version to load (e.g., `yolo11n` for nano, `yolo11s` for small). |
|  | `device` | String | `"auto"` | **Hardware Accel:** Hardware to run inference on. `"auto"` detects CUDA (GPU) first, then defaults to CPU. |
| **AI (IA)** | `ia` | Boolean | `true` | **AI Services:** Master toggle for heavy AI services including face, pose, hand landmarks, and chess detection. |
| **Output** | `publish_visualizations` | List | See file | **Visual Feed:** A list of annotated image streams to publish (e.g., images with bounding boxes or skeleton overlays). |
|  | `publish_data` | List | See file | **Raw Data:** A list of non-image topics to publish, such as raw bounding box coordinates or polygon geometry. |

---

### Implementation Details

* **Visualizations:** The items listed in `publish_visualizations` (like `face_landmarks` or `coco_detections`) determine which `sensor_msgs/Image` topics will be active for tools like **Rviz** and **rqt_image_view**.
* **Data Streams:** The `publish_data` list controls the output of raw mathematical data (e.g., `coco_bboxes`), which is more efficient for other ROS nodes to process than raw images.
* **Location:** This file must be located at `vision_utilities/src/config.yaml` to be correctly parsed by the `vision_utilities.py` node at runtime.

## Execution

**1. Start the camera (if not using Pepper):**

```bash
rosrun usb_cam usb_cam_node

```

**2. Launch Vision Utilities:**

```bash
rosrun vision_utilities vision_utilities.py

```

## Environment Variables

If `llm_mode` is set to `openai` in the config, you must export your API key:

```bash
export GPT_API="your_azure_openai_key_here"

```
